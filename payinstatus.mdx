#### Pay-In Check QR Status

The **Pay-In Check QR Status** endpoint allows clients to verify the status of a QR-based payment and retrieve the related pay-in order details from the database using a `ref_id`. This is typically used after generating a QR code and wanting to confirm if the payment was completed.

---

#### Endpoint

```http
POST /payin/check-qr-status
```
#### Authentication

This endpoint supports two forms of authentication:

1. Client credentials via headers:


-x-client-id: finflex_test_6868706820
-x-client-secret: finflex_test_be3798f8-deed-4215-97b4-556049549670

2.Bearer token (optional alternative):

```bash
Authorization: Bearer <your_access_token>
```
#### Request Body

```bash
{
  "client_id": "finflex_test_6868706820",
  "slug": "QXCtRyzcsJp9",
  "ref_id": "QXCtRyzcsJp91752058602"
}

```

Explanation of Request Fields
-client_id: The Finflex client ID associated with your account.

-slug: The QR slug used to identify the payment.

-ref_id: The unique reference ID for the specific pay-in order you're checking.

#### Success Response (200 OK)0

```
{
  "typof": {
    "payment_status": "completed",
    "amount": 10,
    "transaction_id": "txn_12345678"
  },
  "payin_order": {
    "order_id": "321sdf",
    "amount": 10,
    "status": "completed",
    "ref_id": "QXCtRyzcsJp91752058602",
    "created_at": "2024-07-04T10:30:00Z"
  }
}

```

typof: Response from the Typof API regarding QR payment.

payment_status: Status of the payment (e.g., completed, pending).

amount: The paid amount.

transaction_id: Unique transaction ID generated by the payment processor.

payin_order: The pay-in order record retrieved from the database.

order_id: The internal Finflex order ID.

amount: The amount originally set for the pay-in order.

status: The final status of the pay-in order.

ref_id: The reference ID tied to the pay-in.

created_at: Timestamp of order creation.

#### Error Response (401 Unauthorized)
Returned when authentication credentials are missing or invalid.

```bash
{
  "status": false,
  "message": "Authentication failed - please provide either valid client credentials or a valid Bearer token"
}


```

#### Not Found Response (404 Not Found)

```bash
{
  "code": 404,
  "message": "Payin order not found with the provided ref_id"
}


```
